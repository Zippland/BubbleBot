Writes a file to the filesystem.

**Path handling**:
- `~` refers to session_dir (NOT the system $HOME)
- Relative paths are resolved relative to session_dir
- All paths must be within session_dir; writing files outside will return an error

Usage:
- This tool will overwrite the existing file if there is one at the provided path.
- If this is an existing file, you MUST use the read_file tool first to read the file's contents. This tool will fail if you did not read the file first.
- ALWAYS prefer editing existing files in the codebase. NEVER write new files unless explicitly required.
- NEVER proactively create documentation files (*.md) or README files. Only create documentation files if explicitly requested by the User.
